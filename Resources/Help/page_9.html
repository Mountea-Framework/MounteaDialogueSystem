Production Checklist & Useful Links

Finalize shipping readiness with a repeatable checklist, then keep a maintenance path for future updates.

What You Will Learn

  • How to perform a final pre-release dialogue audit.
  • How to identify high-risk areas for regression testing.
  • Where to find documentation and community support resources.

When To Use This

Use this chapter before milestone builds, QA handoff, and release candidates. Repeat after major engine/plugin updates.

Prerequisites

  • All previous chapters validated at least once.
  • Representative test maps and dialogue scenarios available.
  • Known ownership for fixing issues discovered in the checklist pass.

Step-by-Step Workflow

  1. Run a full path test for critical conversations.
  2. Confirm tags/settings are source-controlled and reproducible.
  3. Check UI readability and interaction controls on all target inputs.
  4. Confirm no fallback pages are shown in tutorial/help sections.
  5. Document known limitations and assign owners.

Validation Checklist

  • All essential conversation paths complete without blockers.
  • No critical warnings in logs during dialogue sessions.
  • Tutorial pages and help links open correctly in editor.
  • Team can onboard from this guide without hidden tribal knowledge.

Common Mistakes + Fixes

Mistake: treating one successful run as production ready.
Fix: validate multiple branches, edge states, and interruption paths.

Mistake: no ownership for follow-up issues.
Fix: assign fix owner and due date for each discovered issue.
