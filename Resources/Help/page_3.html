Gameplay Tags Setup & Verification

Ensure gameplay tags are present, synchronized, and trusted before building branching logic.

What You Will Learn

  • How dialogue tags are sourced and validated.
  • How to detect stale or missing tag data quickly.
  • How tag readiness affects conditions and decorators.

When To Use This

Run through this chapter before authoring conditions. It prevents broken branches caused by nonexistent or misspelled tags.

Prerequisites

  • Mountea project settings already reviewed.
  • Gameplay Tags project settings accessible.
  • Internet available only if pulling remote defaults; otherwise local tags are enough.

Step-by-Step Workflow

  1. Open Gameplay Tags settings and review current list/source files.
  2. Open Mountea settings and verify tags URL / auto-check behavior.
  3. Restart editor if tag source changed.
  4. Open any graph node that depends on tags and validate dropdown/search results.

Validation Checklist

  • Expected dialogue tags are visible in tag pickers.
  • No warnings about missing tag table/source file.
  • Tag-based conditions in sample graph evaluate as expected.

Common Mistakes + Fixes

Mistake: adding tags without reloading dependent editors.
Fix: close/reopen graph editor after tag updates.

Mistake: using free-text names in conditions.
Fix: always pick tags from validated list.
