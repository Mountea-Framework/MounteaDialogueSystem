Advanced Authoring (Decorators, Conditions, Events)

Scale beyond linear conversations using reusable conditions, contextual decorators, and event-driven responses.

What You Will Learn

  • How to organize advanced branches without unreadable graph growth.
  • When to use decorators versus explicit branch nodes.
  • How to test event side effects safely.

When To Use This

Use this chapter after baseline conversation flow is stable. Advanced authoring should extend validated behavior, not replace fundamentals.

Prerequisites

  • At least one working graph with UI feedback.
  • Known gameplay tags and runtime variables available for conditions.
  • Agreement on naming conventions for reusable logic.

Step-by-Step Workflow

  1. Extract repeated checks into reusable condition/decorator logic.
  2. Add event hooks only where gameplay state must change.
  3. Group related branches spatially and name nodes for searchability.
  4. Test every branch end state in PIE with logged outcomes.
Authoring principle:
One node = one clear responsibility.
If a node does too much, split behavior and keep flow explicit.

Validation Checklist

  • All branches reachable by at least one test path.
  • No event side effect triggers twice unexpectedly.
  • Decorators remain understandable to teammates during review.

Common Mistakes + Fixes

Mistake: hidden logic spread across too many decorators.
Fix: keep critical branch gates visually explicit in graph.

Mistake: adding events without rollback plan.
Fix: log side effects and isolate risky actions behind clear conditions.
