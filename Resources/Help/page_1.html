Installation & Plugin Enablement

Confirm the plugin is installed correctly and all required editor/runtime modules are available.

What You Will Learn

  • How to verify plugin activation for the current project.
  • How to confirm data/content folders are available.
  • How to identify installation issues before runtime debugging.

When To Use This

Use this chapter first on every machine where the plugin is used. It prevents late failures caused by missing modules or disabled content.

Prerequisites

  • Project can open in Unreal Editor.
  • You have permission to edit plugins and project settings.
  • Engine version is compatible with current plugin branch.

Step-by-Step Workflow

  1. Open Plugins window and confirm Mountea Dialogue System is enabled.
  2. Restart editor if activation state changed.
  3. Open plugin data and example folders to validate content mount points.
  4. Open Project Settings and ensure Mountea Dialogue category is visible.

Validation Checklist

  • No startup warnings about missing dialogue classes/modules.
  • Project settings section is visible and editable.
  • Plugin sample assets are visible in Content Browser.

Common Mistakes + Fixes

Mistake: plugin enabled but editor not restarted.
Fix: restart editor after any plugin state change.

Mistake: sample content hidden.
Fix: enable plugin content visibility in Content Browser filters.
