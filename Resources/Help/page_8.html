Debugging & Troubleshooting

Use a structured fault isolation approach to locate setup errors fast and prevent regressions.

What You Will Learn

  • How to isolate failures by subsystem: settings, tags, world wiring, graph, UI.
  • How to build quick reproducible tests for common failures.
  • How to document a fix so teams avoid repeated breakages.

When To Use This

Use this chapter whenever dialogue flow stops working after changes, engine updates, or content migrations.

Prerequisites

  • A known failing scenario in PIE or editor preview.
  • Access to Output Log and test content.
  • Ability to compare current setup against a previously working state.

Step-by-Step Workflow

  1. Reproduce failure in the smallest possible scenario.
  2. Confirm settings and tag integrity before inspecting graph details.
  3. Validate participant component placement and runtime ownership.
  4. Inspect UI event chain and close/open lifecycle.
  5. Apply one fix, rerun same scenario, then regress-test a second path.

Validation Checklist

  • Failure is reproducible before fix and gone after fix.
  • No new warning/errors introduced in Output Log.
  • At least one alternative conversation path still works.

Common Mistakes + Fixes

Mistake: changing multiple systems before retest.
Fix: use one-change-per-test iteration.

Mistake: fixing only the symptom in UI layer.
Fix: verify data source and participant lifecycle first.
