Building Your First Dialogue Graph

Create a minimal but complete graph that confirms data, tags, and runtime participants are all integrated.

What You Will Learn

  • How to author a first graph with clear entry, branch, and exit.
  • How to keep graph structure readable from day one.
  • How to run a quick success/fail validation loop.

When To Use This

Use after runtime wiring is ready. This chapter confirms your project can execute an authored graph in PIE.

Prerequisites

  • World setup completed.
  • Gameplay tags validated.
  • At least one actor pair ready for interaction testing.

Step-by-Step Workflow

  1. Create a new dialogue asset.
  2. Add greeting node, one branch condition, and one clean ending node.
  3. Use auto-arrange once to keep readability.
  4. Bind graph to one participant flow and run PIE test.
Minimum first graph shape:
Entry -> Greeting -> Choice/Condition -> Response A/B -> Exit

Validation Checklist

  • Graph compiles without missing references.
  • One full conversation path reaches exit in PIE.
  • Branch behavior is deterministic for your selected condition.

Common Mistakes + Fixes

Mistake: starting with a very large graph.
Fix: validate one tiny graph first, then scale.

Mistake: no explicit exit path.
Fix: always include predictable termination nodes.
