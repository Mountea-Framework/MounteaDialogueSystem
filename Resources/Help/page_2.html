Chapter 3 of 10

Project Settings Deep Dive

Configure editor and runtime defaults so every dialogue graph starts from predictable baseline rules.

What You Will Learn

  • Which settings affect graph authoring experience.
  • How auto-arrange and node options influence daily workflow.
  • How to keep team-wide defaults stable while allowing local overrides.

When To Use This

Use this chapter right after installation, and revisit when graph layout or editor behavior feels inconsistent between teammates.

Prerequisites

  • Plugin enabled and project restarted.
  • Project settings category visible.
  • At least one test dialogue asset for validation.

Step-by-Step Workflow

  1. Open Mountea Dialogue Editor settings.
  2. Review node behavior toggles (rename, automatic names, standard nodes).
  3. Tune auto-arrange values for your graph size profile.
  4. Save config and reopen one graph to validate effect.

Open Mountea Dialogue Settings

Suggested first-pass values:
OptimalHorizontalDistance: 220-300
OptimalVerticalDistance: 100-160
AutoLayoutStrategy: Tree

Validation Checklist

  • Graph auto-arrange no longer causes overlapping core nodes.
  • Node naming behavior matches team expectation.
  • No deprecated-visual warnings are treated as blocking issues.

Common Mistakes + Fixes

Mistake: changing many settings at once and losing baseline.
Fix: change one category, validate, then continue.

Mistake: forcing one layout profile for every graph type.
Fix: select values that serve your most frequent graph size.
