World Setup (GameMode, HUD, Participants)

Wire runtime ownership points so dialogue requests, UI, and participants can communicate in play sessions.

What You Will Learn

  • Which runtime actors/components are mandatory for an end-to-end interaction.
  • How to place participants and verify registration.
  • How world settings impact dialogue startup behavior.

When To Use This

After settings and tags are stable, before graph authoring deep dive. Runtime wiring should be ready before content authoring scales up.

Prerequisites

  • Playable test level available.
  • GameMode and HUD classes can be edited.
  • Player and NPC actors are available for participant setup.

Step-by-Step Workflow

  1. Open World Settings and validate active GameMode for test map.
  2. Configure HUD/UI integration expected by your dialogue flow.
  3. Add required participant component to actors that can speak or receive dialogue.
  4. Run PIE and trigger one interaction from player to NPC.

Validation Checklist

  • Participant component exists on all expected actors.
  • HUD/UI receives dialogue events at runtime.
  • No silent failures when initiating first interaction.

Common Mistakes + Fixes

Mistake: testing in a map with different GameMode than configured setup map.
Fix: verify World Settings per map.

Mistake: participant component added only to NPC.
Fix: confirm both interaction sides are wired where required by your flow.
