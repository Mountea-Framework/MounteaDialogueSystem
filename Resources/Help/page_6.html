Chapter 7 of 10

UI Integration & Participant UI Flow

Connect authored graph output to UI components with reliable participant-driven updates.

What You Will Learn

  • How UI receives dialogue state and text updates.
  • How participant UI component participates in flow ownership.
  • How to test UI visibility, transitions, and cleanup states.

When To Use This

Use after first graph runs in PIE without UI polish. This chapter makes dialogue user-facing and testable by designers.

Prerequisites

  • Working graph from Chapter 6.
  • UI widget blueprint/class ready for dialogue display.
  • Participant component configuration understood.

Step-by-Step Workflow

  1. Bind dialogue events to UI show/hide and text population logic.
  2. Ensure participant UI component references expected widget owner.
  3. Test interaction start, mid-conversation choice, and completion cleanup.
  4. Run test with both keyboard/controller to validate interaction UX.

Open World Settings

Validation Checklist

  • Dialogue panel appears exactly once per interaction.
  • Speaker/line updates are synchronized with graph events.
  • UI closes cleanly at dialogue end or cancel path.

Common Mistakes + Fixes

Mistake: multiple widget instances per interaction.
Fix: centralize create/show ownership and guard against duplicate spawn.

Mistake: UI stays visible after exit node.
Fix: bind explicit cleanup on dialogue finish and interruption paths.
